Iconography

orion-icon fonts

Orion icon fonts are generated during the build process

The icon fonts are sourced from SVG files in /assets/16px and assets/24px

Icon font usage:

<i class="orion-icon orion-icon-[name]"></i>

Example:

<i class="orion-icon orion-icon-alert"></i>


Icon sizing:

Size can be set by the font-size style on the parent container or on the icon element explicitly

Example:

<p style="font-size:30px"><i class="orion-icon orion-icon-alert"></i></p>


Icon color:

Color can be set by the color style on the parent container or on the icon element explicitly

Example:

<i style="color:red" class="orion-icon orion-icon-alert"></i>


Icon rotation:

Can be done by the app developer using custom styles or leveraging the following orion utility classes:

rotate-45, rotate-90, rotate-180

Example:

<i class="orion-icon orion-icon-teen rotate-90"></i>


Icon flipping:

Can be done by the app developer using custom styles or leveraging the following orion utility classes:

flip-x, flip-y

Example:

<i class="orion-icon orion-icon-teen flip-y"></i>

 Layout grid

Layout grid

The layout grid is a 12-column grid that is used to create consistent layouts. It is based on the Bootstrap grid.

The grid is responsive and will automatically adjust to the size of the viewport. The grid is built using CSS flexbox. Use the layout grid classes to create a responsive grid layout.

For viewports 768px and smaller (XS and SM), mobile patterns will be used.

For viewports 769px and larger (SM, MD, LG and XL), desktop patterns will be used.

The grid is built using containers. The container class .orion-container is used to create a container that will center the grid and add padding to the left and right of the grid. The container class is required for the grid to work.

The row class .orion-row is used to create a row that will contain the columns. The row class is required for the grid to work.

The column classes are used to create the columns. The column classes are required for the grid to work. The column classes are named using the format .orion-col-breakpoint-number of columns.

Breakpoints:
  • XS: 414px
  • SM: 744px
  • MD: 1024px
  • LG: 1440px
  • XL: 1920px
Containers:
  • XS: 368px
  • SM: 656px
  • MD: 944px
  • LG: 1320px
  • XL: 1792px

Links

orion-link

Link is an accessible element used for navigation. It is also known as Anchor, <a>, Text Link or Hyperlink . This component is styled to resemble a hyperlink and semantically renders an <a>. Links are visually similar to tertiary buttons, but are semantically different, which matters for accessibility. Use a link when you want to navigate the user to a different location. Use a button when you want to cause a change in either back-end or the front-end of the website. (A "Log in" link will navigate the user to a form, a "Log in" button will submit the form and log the user into the application.)

Using an <a> tag is an acceptable way to render a text link, but a component is also available with additional options.

All links can have a subtle quiet style, without an underline. This style should only be used when the placement and context of the link is explicit enough that a visible underline is not necessary. Color alone is not enough context to meet accessibility requirements. Commonly used in footer navigation.

A link can be disabled by removing the href or adding the disabled prop.
